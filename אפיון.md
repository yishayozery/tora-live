# TORA_LIVE — אפיון מוצר מלא v2.0
**פלטפורמת שיעורי תורה אונליין לרבנים**
*עודכן: אפריל 2026*

---

## חזון המוצר

פלטפורמה שנותנת לרבנים שליטה מלאה על הנוכחות הדיגיטלית שלהם — לוח זמנים, שידורים, ארכיון שיעורים, ותקשורת עם תלמידים — ממקום אחד. הציבור מקבל גישה חופשית לצפייה, תלמידים רשומים יכולים לתקשר עם הרב.

---

## משתמשים

| סוג | גישה | תיאור |
|-----|------|--------|
| אדמין | מייל ב-.env כניסה לממשק ניהול | מאשר רבנים, מנהל תוכן, רואה כל הנתונים |
| רב | נרשם, ממתין לאישור אדמין | מנהל דף, שיעורים, יומן, צ'אט |
| תלמיד רשום | נרשם חופשי | יכול לשלוח שאלות בצ'אט, ניתן לחסום |
| צופה אנונימי | ללא הרשמה | צפייה בשיעורים בלבד |

---

## זרימת משתמשים

רב חדש: רב ממלא טופס הרשמה, אדמין מקבל התראה במייל, אדמין מאשר/דוחה, רב מקבל מייל, רב מתחבר ומגדיר דף.

תלמיד חדש: תלמיד נרשם (מייל + סיסמה), כניסה מיידית, יכול לשאול שאלות בצ'אט.

חסימת תלמיד: רב / אדמין לוחצים "חסום", התלמיד רואה הודעה ברורה, לא יכול לשלוח הודעות בשום מקום.

## מודולים ראשיים

### 1. דף הרב

כתובת: tora-live.co.il/rabbi/[slug]

מכיל:
- תמונה, שם, תיאור חופשי (הרב כותב כרצונו, ללא הגבלת תוכן מצד הפלטפורמה)
- קטגוריות שיעורים שהרב יצר בעצמו
- דשבורד מיני ציבורי: סך שעות שיעור, מספר שיעורים, מספר צופים כולל
- לוח שיעורים עתידיים
- כפתור "שלח שאלה" לתלמידים רשומים בלבד
- כל שיעור: כותרת, תיאור, קישורים לפלטפורמות, דף מקורות PDF, תמלול

---

### 2. ניהול לוח שנה — סנכרון דו-כיווני

פלטפורמות נתמכות:
- Google Calendar
- Microsoft Outlook / Office 365
- Apple iCal (CalDAV)
- אפשרות לחבר מספר יומנים במקביל

הגדרות סנכרון שהרב בוחר:
- כיוון: חד-כיווני או דו-כיווני
- כשמוסיף שיעור ב-TORA_LIVE: יצירה אוטומטית ביומן
- כשמוסיף אירוע ביומן: ייבוא אוטומטי ל-TORA_LIVE לפי תגית
- כשמבטל שיעור: עדכון אוטומטי בשני הכיוונים

---

### 3. ניהול שיעורים

שדות:
- כותרת + תיאור חופשי
- קטגוריה (הרב יוצר קטגוריות משלו)
- תאריך ושעה
- קישורים: YouTube, Spotify, Apple Podcasts, SoundCloud, כתובת אחרת
- העלאת דף מקורות (PDF)
- האם לסנכרן ליומן? (כן/לא)

---

### 4. שידור חי

הרב פותח שידור ב-YouTube / Zoom / כל פלטפורמה, מדביק לינק ב-TORA_LIVE.
האתר מציג embed + צ'אט שאלות בצד.

---

### 5. צ'אט שאלות

- צופה אנונימי: רואה צ'אט, לא יכול לכתוב
- תלמיד רשום: יכול לשלוח שאלות
- תלמיד חסום: רואה צ'אט, מקבל הודעה "אינך מורשה לשלוח הודעות"
- הרב: רואה שאלות, מסמן "נענה", חוסם משתמש ישירות

---

### 6. תרומות — לזכר / לזכות

מטרה: תמיכה בהמשך פעילות הפלטפורמה.

זרימה: לוחצים "תרום", בוחרים סכום, מוסיפים שם לזכר/לזכות, תשלום, קבלה PDF במייל.

שדות:
- סכום (חופשי או מוגדר מראש)
- לזכר / לזכות + שם (אופציונלי)
- מייל + שם התורם לקבלה
- האם להופיע בלוח כבוד? (כן/לא)

סליקה: Tranzila / Cardcom (ישראלי)
קבלה: PDF אוטומטי נשלח למייל
הצגה: לוח כבוד עם שמות תורמים שבחרו להופיע, ללא סכומים.

---

### 7. דשבורד ציבורי — עמוד הבית

מדדים לציבור:
- סך שעות לימוד באתר
- סך שיעורים שעלו
- סך צפיות כוללות
- מספר רבנים פעילים (העלו שיעור ב-30 יום אחרונים)
- שידורים חיים עכשיו
- שיעורים מתוכננים השבוע

כל הנתונים מתעדכנים בזמן אמת.

---

### 8. דשבורד הרב — ממשק פרטי

- צפיות לפי שיעור ולפי תקופה
- שאלות שנשאלו בצ'אט
- שיעורים שעלו לפי חודש
- שעות תוכן כולל
- שיעורים הבאים ביומן
- פניות צור קשר שממתינות לתשובה

---

### 9. צור קשר עם הרב

- כפתור "שלח שאלה" — רק לתלמידים רשומים
- הפנייה מגיעה לממשק הרב (המייל/טלפון של הרב לא נחשף)
- הרב עונה מתוך הממשק
- התלמיד מקבל תשובה במייל + בממשק

---

### 10. מערכת דיווח

- כפתור "דיווח" בכל שיעור ובכל הודעת צ'אט
- קטגוריות: תוכן לא הולם / ספאם / בעיה טכנית
- אדמין מקבל התראה במייל
- אדמין יכול: להסיר שיעור, לחסום רב, לחסום תלמיד

---

### 11. תמלול אוטומטי (Phase 4)

- Whisper API (OpenAI)
- הרב יכול לערוך לפני פרסום
- מוצג מתחת לסרטון
- ניתן לאינדוקס בגוגל — ערך SEO גבוה

## ממשק אדמין

כניסה: מייל מוגדר ב-.env מפנה לדף /admin

פעולות:
- אישור רבנים: רשימת ממתינים + אישור/דחייה
- ניהול תלמידים: חסימה, הסרת חסימה
- ניהול שיעורים: הסרת שיעור בעייתי
- דיווחים: ניהול תלונות
- נתוני האתר: סטטיסטיקות מלאות כולל מספר רבנים רשומים
- תרומות: רשימת תרומות + ייצוא לאקסל

## ארכיטקטורה טכנית

Stack:
- Frontend: Next.js 14+ (App Router) + TypeScript + Tailwind CSS
- Backend: Next.js API Routes
- Database: PostgreSQL (Supabase)
- Auth: NextAuth.js (מייל+סיסמה)
- Storage: Cloudflare R2 (PDFs, תמונות)
- Calendar: Google Calendar API + Microsoft Graph API + CalDAV
- Video: YouTube embed
- Podcast: RSS Feed generation
- AI: OpenAI Whisper API (תמלול)
- סליקה: Tranzila / Cardcom API
- קבלות: PDFKit
- Deploy: Vercel

## מסד נתונים (Prisma Schema)

Rabbi:
  id, slug, name, bio (Text), photo, email, isApproved, isBlocked, calendarSync (Json), createdAt

Student:
  id, email, name, isBlocked, blockedReason, createdAt

Lesson:
  id, rabbiId, title, description (Text), category, scheduledAt, durationMin,
  youtubeUrl, spotifyUrl, applePodcastUrl, otherUrl, sourcesPdfUrl,
  transcriptText (Text), isLive, liveEmbedUrl, viewCount, calendarEventId, createdAt

ChatMessage:
  id, lessonId, studentId, content, isAnswered, createdAt

ContactRequest:
  id, rabbiId, studentId, message (Text), reply (Text), status (pending/replied), createdAt

Donation:
  id, amount, donorName, donorEmail, dedicationName, dedicationType (לזכר/לזכות),
  showPublicly, receiptSent, paymentRef, createdAt

Report:
  id, lessonId, reporterEmail, category (inappropriate/spam/technical), description, status (open/resolved), createdAt

## Phases — סדר פיתוח

Phase 1 - MVP (6-8 שבועות):
- הרשמת רב, אישור אדמין
- הרשמת תלמיד (מנגנון זיהוי)
- דף פרופיל ציבורי לרב
- העלאת שיעור + קישורים לפלטפורמות
- קטגוריות שיעורים
- העלאת דף מקורות PDF
- דשבורד ציבורי בעמוד הבית
- דשבורד רב פרטי
- ממשק אדמין (אישור רבנים, חסימה, נתונים)
- כניסת אדמין דרך מייל ב-.env

Phase 2 - Calendar and Chat (4 שבועות):
- חיבור Google Calendar דו-כיווני
- חיבור Outlook / iCal
- הגדרות סנכרון לפי רב
- צ'אט שאלות בשידור חי
- מערכת צור קשר (תלמיד לרב)
- חסימת תלמידים

Phase 3 - Donations (3 שבועות):
- אינטגרציה Tranzila / Cardcom
- טופס תרומה עם שדה לזכר/לזכות
- קבלה PDF + שליחה במייל
- לוח כבוד תורמים

Phase 4 - AI (3 שבועות):
- תמלול אוטומטי Whisper
- עריכת תמלול ידנית
- RSS Feed לפודקאסט

## כללי פיתוח לסוכנים

1. RTL תמיד - כל טקסטים בעברית עם dir="rtl"
2. Mobile First - רוב הצופים מנייד
3. אבטחה - מייל/טלפון של רבנים לא נחשפים לציבור לעולם
4. נגישות - WCAG 2.1 AA
5. ביצועים - Lazy loading לסרטונים ו-PDFs
6. חסימה - תלמיד חסום מקבל הודעה ברורה, לא שגיאה טכנית
